# chap 06_5 정리 
26.02.24 update

## 06_5_1~4
* mmap: mmap은 메모리 특정 공간에 파일을 매핑함 -> 해당 파일을 읽기 위해서 저장매체에 접근하는 것이 아닌, 메모리의 데이터에 접근하는 방식
* mmap으로 디스크의 파일 read/write 시에는 시스템 호출과 데이터 복사가 주는 부담이 없음 
* but mmap은 프로세스 주소 공간과 파일의 사상관계 유지 위해서 특정 데이터 구조 사용해야함 -> 성능에 부담을 가져옴 
* mmap 호출 시 매개변수가 MAP_SHARED면 사상된 영역의 변경 내용은 디스크 파일에 직접 기록됨 
* 해당 파일의 크기가 실제 물리 메모리의 크기보다 커도 괜찮음 

## 06_5_5~
* 동적 링크 라이브러리: 많은 프로세스가 동일한 동적 링크 라이브러리에 의존 -> mmap으로 해당 라이브러리를 사용하는 모든 프로세스의 주소 공간에 직접 매핑 가능 
* strace 명령어: 프로그램 실행과정과 관련된 모든 시스템 호출을 출력함
