## 06_2_1
* 디스크 입출력 처리, cpu 실행 작업은 서로 의존하지 않는 독립적 작업 (입출력 처리 시 cpu 개입 x)
* 장치 제어기(device controller) 
    * 디스크와 같은 입출력 장치에서 기계 부분을 제외한 나머지 전자 부분 
    * 초창기에는 전자 부분 역할이 간단했으나, 마이크로 컴퓨터 시스템으로 발전하면서 자체적인 프로세서 & 펌웨어를 갖추게 됨 => cpu가 도와주지 않는 상황에서도 복잡한 작업을 할 수 있음 
        * 자신만의 버퍼나 레지스터를 갖고 있음 -> 장치에서 읽은 데이터나 장치에 저장할 데이터를 저장 가능 
    * != 장치 드라이버(device driver)
        * 얘는 운영 체제에 속한 코드임. 장체 제어기는 장치 드라이버에서 명령을 받아 외부 장치를 제어하는 하드웨어에 해당함 

## 06_2_2~4
* 직접 메모리 접근(direct memory access, DMA) 
    * cpu의 개입이 없는 상황에서 직접 장치와 메모리 사이에 데이터를 전송할 수 있는 작동 방식 
    * cpu는 어떻게 데이터를 복사할지 알려주는 명령어를 DMA에 전달해야함 (어디에서 어디록 기록할지, 얼마나 기록할지, 어느 메모리 위치에서 읽어야하는지, 어떤 장치에서 읽고 쓸지) -> 그 후에 DMA에 작업 시작 가능 
    * bus arbitration(버스 중재), 버스의 사용 권한을 요청 후 이어서 장치를 작동시킴 
        * 장치의 제어기의 버퍼에서 데이터 읽고 -> DMA가 지정된 메모리 주소에 데이터를 쓰는 방식으로 데이터 복사 완료
    * 운영체제가 DMA에 필요한 가상 주소와 물리 메모리 주소 사이의 사상 정보를 제공 -> dma가 가상 주소를 기반으로 직접 데이터를 전송할 수 있게 됨 
    * 캐시, 메모리와 불일치 문제? : DMA가 데이터 전송을 완료하면 인터럽트 작동 방식을 통해 CPU에 알려줌으로써 해결 가능
