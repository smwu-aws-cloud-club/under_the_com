# chap 05_3 정리 
26.02.11 update

## 05_3_1~
* `cache line`: 해당 데이터가 있는 곳의 '묶음' 데이터를 캐시에 저장하는 것 
    * 일반적으로 64 byte 

### 캐시 성능 방해 요소 
* `cache ping-pong 현상` 
    * 책 속 예시 참고: 캐시 일관성 보장을 위해서 다중 스레드 다중 코어의 각 캐시에 서로의 a 변수를 invalidation 해야함 (캐시 튕김 현상)
* `false sharing 현상`
    * 비록 스레드 두개가 어떤 데이터도 공유하지 않는 것처럼 보여도 캐시의 동작 방식에 따라 캐시 라인을 공유할 가능성이 있음 => 캐시 튕김 문제 발생시킴
