## 05_2_1~3
* 프로그램 지역성 
    * temporal locality -> 높을수록 캐시 친화적
    * spatial locality -> 높을수록 캐시 친화적
* 메모리 풀 기술은 일반적으로 고성능 요구 사항이 있을때만 사용됨 (malloc, new)
    * malloc을 사용해서 할당 받으면 메모리 조각 N개가 힙 영역 이곳저곳 흩어져 있음 => 공간성 지역성 bad 
    * instead 메모리 풀 기술은 커다란 메모리 조각을 미리 할당받음 -> 이후 메모리를 요청하거나 해제할때 malloc 거치지 x => very cache 친화적 
* 5.2.3의 코드 예시 참고 

## 05_2_4~6
* arr을 다른 구조체에 넣고 List 구조체 안에 이 구조체를 가리키는 포인터를 추가 가능 
    * arr은 cold data, next pointer/value는 hot data 
* 지역성 원칙 관점에서는: std::vector 컨테이너 사용 > std::list 컨테이너
    * 배열은 하나의 연속된 메모리 공간에 할당되지만, 연결 리스트는 이곳저곳에 흩어져있을 수 있음 
* 반드시 성능 분석 도구 사용하여 캐시 적중률이 시스템 성능의 병목이 되는지 먼저 판단이 필요 
    * 병목 아니면 굳이 최적화에 힘쏟지마
