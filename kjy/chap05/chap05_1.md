# chap 05_1 정리

## 05_1_1~2
* cannikin law: cpu와 메모리의 속도가 같아야 가장 좋은 성능을 발휘 가능하다 
    * 그러나 현실은 x -> 시간이 지날수록 속도 차이가 더 벌어지고 있음 
* 이러한 문제를 캐시를 활용하여 어느정도 해소 가능 
    * x86의 최신 CPU와 메모리 사이에는 L1, L2, L3 캐시가 존재 
    * 레지스터 속도 > L1 > L2 > L3 
* CPU가 메모리에 접근하고자 할때 -> 먼저 L1 캐시 -> L2 캐시 -> L3 캐시 -> 메모리 접근 순 


## 05_1_3~4
* 캐시 갱신 
    * 캐시와 메모리 데이터가 불일치 
    * Write-through: 캐시 갱신 시 메모리도 같이 갱신 (동기적 방식)
    * Write-back: 캐시에서 제거된 데이터가 수정된 적이 있으면 메모리에 갱신 (비동기적 방식)
* 다중 코어 캐시의 일관성을 유지하여 설계해야함 

## 05_1_5~8
* 디스크의 캐시는 메모리 (not register)
* 일부 프로세스에서 자주 사용하지 않은 메모리 데이터를 디스크에 기록 -> 이 데이터가 차지하던 물리 메모리 공간 해제 가능 
* 프로세스의 데이터는 임의로 디스크에 보관되어 있을 수 있으므로 주의 
* 분산 파일 시스템
